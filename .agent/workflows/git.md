이 워크플로우는 프로젝트의 이력을 투명하게 관리하기 위해 변경 사항을 분석하고, **Conventional Commits** 규격에 맞는 메시지 작성을 지원합니다. 시니어 엔지니어의 관점에서 변경 사항이 무엇인지 명확히 드러나도록 메시지를 구성하는 것이 핵심입니다.

### 📋 변경 사항 요약 (Summary of Changes)
// turbo
1. 현재 작업 디렉토리의 변경 사항(`git status`)을 확인하고, 주요 변경 필드 및 메서드, 기능적 차이점을 사용자에게 보고합니다.

### 📝 커밋 메시지 제안 (Proposed Commit Message)
2. 분석된 내용을 바탕으로 다음과 같은 접두사가 포함된 메시지를 제안합니다.
   - `feat`: 새로운 기능 추가
   - `fix`: 버그 수정
   - `docs`: 문서 수정 (README.md, architecture.md 등)
   - `style`: 코드 포맷팅 (로직 변경 없음)
   - `refactor`: 가독성 또는 유지보수성 향상을 위한 코드 구조 변경
   - `test`: 테스트 코드 추가 및 수정
   - `perf`: 성능 향상

### 🚀 스테이징 및 커밋 (Stage & Commit)
3. 변경 사항을 스테이징(`git add .`)하고, 확정된 메시지로 커밋을 실행합니다.

### 📤 원격 저장소 푸시 (Push to Remote)
4. 최종 커밋 완료 후, 사용자에게 원격 저장소로 `push`할지 확인 받은 후 명령을 실행합니다.
