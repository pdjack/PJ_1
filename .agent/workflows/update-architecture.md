이 워크플로우는 새롭게 구현된 기능을 감지하거나 사용자로부터 입력받아 아키텍처 문서를 최신 상태로 유지하는 과정을 안내합니다. 시니어 엔지니어의 관점에서 시스템의 확장성과 유지보수성을 고려하여 문서를 관리합니다.

### 📝 업데이트 세부 지침 (Rules)
- **관계 정의 중심**: 단순히 기능을 나열하는 대신, 해당 기능이 참조하는 매니저와 사용하는 데이터(SO)를 명확히 기술합니다.
- **Mermaid 다이어그램 필수 업데이트**: 시스템 상호작용이 발생하는 경우 [3. 상호작용] 섹션의 다이어그램 노드를 반드시 추가/수정합니다.
- **데이터 흐름 추적**: 새로운 데이터 구조(SO 등)가 도입되었다면 [4. 데이터 계층 구조]에 명시합니다.
- **Self-Cleaning**: '5. 앞으로 완성해야 할 부분'에 있던 항목이 구현되었다면, 해당 항목을 삭제하고 본문에 내용을 반영합니다.

### 1단계: 추가된 기능 확인
// turbo
1. 최근 변경된 파일이나 구현된 새로운 기능을 확인합니다. 만약 특정 기능에 대해 명시하고 싶다면 지금 말씀해 주세요.

### 2단계: 아키텍처 문서 분석
2. `directives/Architecture/architecture.md` 파일을 읽어 현재 구조를 파악합니다.

### 3단계: 업데이트 내용 제안
3. 위 **[업데이트 세부 지침]**에 따라 문서의 적절한 섹션(기능 설명, 다이어그램, 계층 구조 등)에 배치하여 업데이트안을 작성합니다.
   - **기능 설명 추가**: 역할과 의존성(Manager, SO)을 명시합니다.
   - **시각화 유지**: Mermaid 코드를 수정하여 전체 시스템 구조의 무결성을 유지합니다.

### 4단계: 사용자 승인 및 적용
4. 제안된 변경 사항을 사용자에게 보여주고 승인을 받습니다. 승인 후 파일에 최종 반영합니다.
