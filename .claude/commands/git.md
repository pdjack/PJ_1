변경 사항을 분석하여 의미 있는 커밋 메시지와 함께 Git 커밋 및 푸시를 진행하는 워크플로우입니다.

프로젝트 이력을 투명하게 관리하기 위해 변경 사항을 분석하고, **Conventional Commits** 규격에 맞는 메시지를 작성합니다.

### 1단계: 변경 사항 요약
1. `git status`와 `git diff`로 현재 변경 사항을 확인하고, 주요 변경 필드·메서드·기능적 차이점을 사용자에게 보고합니다.

### 2단계: 커밋 메시지 제안
2. 분석된 내용을 바탕으로 다음 접두사 중 적절한 것을 선택하여 메시지를 제안합니다:
   - `feat`: 새로운 기능 추가
   - `fix`: 버그 수정
   - `docs`: 문서 수정 (README.md, architecture.md 등)
   - `style`: 코드 포맷팅 (로직 변경 없음)
   - `refactor`: 가독성 또는 유지보수성 향상을 위한 코드 구조 변경
   - `test`: 테스트 코드 추가 및 수정
   - `perf`: 성능 향상

### 3단계: 스테이징 및 커밋
3. 사용자 승인 후 변경 사항을 스테이징하고 확정된 메시지로 커밋을 실행합니다.

### 4단계: 원격 저장소 푸시
4. 커밋 완료 후 원격 저장소로 push할지 사용자에게 확인 받은 후 실행합니다.
