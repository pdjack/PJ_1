신규 구현된 기능을 분석하여 알맞은 문서를 찾거나(Find) 새로 생성하여(Create) 정리하는 워크플로우입니다.

구현된 기능이 프로젝트의 어느 부분에 속하는지 판단하고, 기존 문서 업데이트 또는 신규 기능 명세서 생성을 통해 프로젝트의 '공유 메모리'를 최신 상태로 유지합니다.

### 업데이트 세부 지침
- **문서 탐색 및 판단:** `Directives/Architecture/document-map.md`를 참고하여 최적의 대상 문서를 선택합니다. 기존 범주에 속하지 않는 대규모 신규 시스템이면 새 MD 파일을 생성합니다.
- **관계 정의 중심:** 단순 나열 대신 해당 기능이 참조하는 매니저와 사용하는 데이터(SO)를 명확히 기술합니다.
- **Mermaid 다이어그램 업데이트:** 시스템 상호작용이 발생하면 `architecture.md`의 다이어그램을 반드시 업데이트합니다.
- **데이터 흐름 추적:** 새로운 데이터 구조(SO 등)가 도입되었다면 `Directives/Architecture/data-structures.md`에 명시합니다.
- **문서 지도 최신화:** 새로운 문서가 생성되었다면 `Directives/Architecture/document-map.md`를 우선 업데이트합니다.

### 1단계: 추가된 기능 확인
1. 최근 변경된 파일이나 구현된 새로운 기능을 확인합니다. 특정 기능을 명시하고 싶다면 말씀해 주세요.

### 2단계: 관련 문서 및 대상 분석
2. `Directives/Architecture/document-map.md`를 읽어 문서 체계와 각 문서의 역할을 파악합니다.
3. 구현된 기능에 가장 적합한 **기존 문서**를 선정하거나, 필요시 **새로운 문서**를 생성할지 결정합니다.

### 3단계: 업데이트 내용 제안
4. 세부 지침에 맞춰 문서의 적절한 섹션에 업데이트안을 작성합니다.
   - architecture.md의 코드↔기획 매핑 테이블 변경 시 반드시 반영합니다.
   - Mermaid 다이어그램을 수정하여 전체 시스템 구조의 무결성을 유지합니다.
   - 시스템적으로 개선 가능한 부분이 있다면 사용자에게 제안합니다.

### 4단계: 사용자 승인 및 적용
5. 변경 사항을 사용자에게 보여주고 승인을 받은 후 파일에 최종 반영합니다.
