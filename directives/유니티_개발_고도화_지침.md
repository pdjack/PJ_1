# 🚀 유니티 개발 고도화 및 성능 최적화 지침 (Directive)

이 문서는 Antigravity 에이전트가 유니티 프로젝트 내에서 단순 구현을 넘어 시니어 급의 설계와 최적화를 수행하기 위한 지침입니다.

## 1. 데이터 주도 설계 (Data-Driven Design)
*   **ScriptableObject 활용**: 하드코딩된 상수나 단순 데이터를 코드에 남기지 않습니다. 게임 설정, 아이템 능력치, 이벤트 채널 등을 ScriptableObject로 분리하여 기획적 유연성을 확보합니다.
*   **이벤트 기반 아키텍처**: 객체 간 결합도(Coupling)를 낮추기 위해 `UnityEvent`나 ScriptableObject를 이용한 "Game Event" 시스템을 우선적으로 제안합니다.

## 2. 메모리 및 성능 최적화 (Advanced Optimization)
*   **Garbage Collection (GC) 관리**:
    *   `Update()` 내에서의 문자열 연결(String Concatenation), 신규 배열 선언을 엄격히 통제합니다.
    *   `StringBuilder` 또는 캐싱된 변수 사용을 권장합니다.
*   **오브젝트 풀링 (Object Pooling)**: 빈번하게 생성/파괴되는 오브젝트(총알, 파티클, UI 요소 등)는 반드시 유니티 내장 `UnityEngine.Pool` 또는 커스텀 풀링 시스템을 적용합니다.
*   **드로우 콜 (Draw Call) 최적화**: 
    *   UI 구현 시 동일한 텍스처를 사용하는 요소들은 최대한 계층 구조를 묶어 배칭(Batching)이 깨지지 않게 설계합니다.
    *   Pixel Perfect 옵션이나 불필요한 Raycast Target은 비활성화합니다.

## 3. 에디터 생산성 향상 (Editor Automation)
*   **Custom Editor 작성**: 복잡한 인스펙터 설정이 필요한 컴포넌트는 `[CustomEditor]`를 통해 직관적인 인터페이스를 제공하여 개발 효율을 높입니다.
*   **Attributes 활용**: `[Header]`, `[Space]`, `[Range]`, `[Tooltip]` 등을 적극 활용하여 인스펙터 상에서 실수 없이 작업할 수 있는 환경을 만듭니다.

## 4. 디버깅 및 테스트 (Testing Strategy)
*   **Conditional Logging**: 로그 출력 시 `#if UNITY_EDITOR` 또는 `[Conditional("UNITY_EDITOR")]`를 활용하여 빌드 시 로그가 포함되지 않게 관리합니다.
*   **자동화 테스트**: 로직 검증이 필요한 핵심 모듈은 `Unity Test Framework`를 활용한 PlayMode/EditMode 테스트 작성을 고려합니다.

## 5. 작업 프로세스
1.  **설계 분석**: 요청받은 기능의 데이터 구체화 -> 아키텍처 제안 (SO vs Singleton 등).
2.  **성능 검토**: 해당 기능이 매 프레임 실행되는지, 가비지를 생성하는지 사전 체크.
3.  **적응적 구현**: `유니티_개발_규칙.md`를 준수하며 코드를 작성.
4.  **검증**: 에디터 상에서의 동작 확인 및 메모리 할당 여부 체크 제안.
