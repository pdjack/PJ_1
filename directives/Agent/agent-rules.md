# 🤖 AI 에이전트 작업 환경 및 운영 원칙 (Agent SOP)

이 문서는 AI와 함께 개발할 때 지켜야 하는 **협업 방식**에 대한 지침입니다.

## 1. 3단계 아키텍처 (3-Layer Architecture)
AI는 다음 세 가지 단계를 거쳐 작업을 수행합니다:

1. **지시서 (Directive - "무엇을 할까?")**
   - `Directives/` 폴더에 마크다운 파일로 작성됩니다.
   - 목표와 순서, 사용할 도구 등을 자연어로 정의합니다.

2. **지휘관 (Orchestration - "어떻게 할까?")**
   - AI 에이전트 자신입니다. 지시서를 읽고 순서대로 작업을 진행합니다.
   - 오류가 생기면 고치거나 다시 물어봅니다.

3. **실행기 (Execution - "진짜로 하기")**
   - `Execution/` 폴더의 파이썬 스크립트나 코드 수정을 의미합니다.
   - 반복적이고 확실한 작업을 처리합니다.

## 2. 운영 원칙
- **도구가 있는지 먼저 확인하기**: 새로운 기능을 만들기 전에 이미 만들어진 도구(`Execution/`)가 있는지 체크합니다.
- **오류는 배움의 기회**: 무언가 고장 나면 고치고, 왜 그랬는지 지시서(`Directives/`)를 업데이트하여 같은 실수를 하지 않습니다.
- **지시서 업데이트**: 더 좋은 방법이나 규칙이 생기면 지시서를 계속 최신 상태로 유지합니다.
- **선 계획 후 보고 및 개선 제안**: 단순히 코드만 작성하는 것이 아니라, 전체적인 시스템 설계를 분석하고 추가로 개선할 수 있는 부분(리팩토링, 성능 최적화, 보안 등)이 있다면 작업을 시작하기 전 사용자에게 반드시 보고하고 승인을 받습니다.
- **문서 중심 소통 및 공식 보고**: 작업의 플랜(Plan), 과정 설명(Walkthrough), 최종 분석 결과 등 모든 주요 단계는 사용자에게 단순 텍스트가 아닌 마크다운 문서(Artifact) 형식으로 정리하여 명확하게 전달합니다. 이는 작업의 투명성을 높이고 추후 히스토리 관리를 용이하게 하기 위함입니다.

## 3. 파일의 역할 (Organization)
- `.tmp/`: 작업 중에 잠깐 쓰는 파일들 (백업이 필요 없는 파일).
- `Execution/`: 실제 작업을 수행하는 도구들 (파이썬 스크립트 등).
- `Directives/`: 작업 순서와 규칙이 적힌 문서들.
- `Environment/`: `.env` 파일 등 API Key 같은 비밀번호 정보 (공개되면 안 되는 정보).

## 5. AI 활용 가이드: 유니티 편 (AI Workflow Guide)
프로젝트 내에서 AI에게 아래와 같은 업무를 구체적으로 지시할 수 있습니다.

- **데이터 튜닝**: "Slime 몬스터의 공격력을 20% 상향해줘." (AI가 해당 SO를 찾아 수정)
- **씬 분석**: "지금 씬에 있는 매니저들 목록을 보여주고, 없는 게 있다면 생성해줘."
- **오브젝트 배치**: "플레이어 주변에 적 생성 구역(SpawnZone) 3개를 랜덤하게 배치해줘."
- **디버깅**: "현재 콘솔에 떠 있는 에러 메시지를 읽고, 어떤 스크립트가 문제인지 분석해서 고쳐줘."

## 6. 요약
AI 에이전트는 사용자의 의도를 읽고, 정해진 규칙에 따라 분석하며, 정확하게 작업을 수행하기 위해 최선을 다합니다. 모든 작업은 Unity MCP를 통해 실시간으로 조율되므로 언제든 작업 내용을 다시 확인할 수 있습니다.
