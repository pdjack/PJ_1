# 🎮 유니티 개발 기본 규칙 및 코딩 스타일

이 문서는 우리 프로젝트의 일관성과 성능을 위해 반드시 준수해야 하는 **가장 기본이 되는 약속**입니다.

## 1. 이름 짓기 (Naming)
* **대문자 시작(PascalCase):** 클래스 이름, 메서드(함수) 이름, public 변수에 사용합니다. (예: `PlayerController`, `MovePlayer()`)
* **소문자 시작과 밑줄(_camelCase):** private 변수에는 앞에 `_`를 붙여 사용합니다. (예: `_moveSpeed`)
* **인스팩터 노출([SerializeField]):** private 변수를 유니티 화면에서 수정하고 싶을 때 사용하며, 그룹별로 `[Header("이름")]`을 달아 보기 좋게 정리합니다.

## 2. 성능 지키기 (Optimization)
* **Update() 안에서 무거운 기능 금지:** 매 초마다 수십 번 실행되는 `Update()` 함수 안에서 `GetComponent`, `Find` 같이 컴퓨터에 부담을 주는 코드는 절대 쓰지 않습니다.
* **꼬표(Tag) 비교:** `if (other.tag == "Player")` 대신 속도가 더 빠른 `if (other.CompareTag("Player"))`를 사용합니다.
* **비어있는지 확인(Null Check):** 되도록 `if (obj == null)` 처럼 명확하게 코드를 작성합니다.

## 3. 구조와 연결
* **필수 컴포넌트 자동 추가:** 특정 기능이 꼭 필요한 스크립트에는 상단에 `[RequireComponent(typeof(애자_타입))]`을 추가합니다.
* **딱 하나만 있는 관리자(Singleton):** 전체 게임에서 하나만 있어야 하는 관리자(Manager) 클래스는 정해진 싱글톤 규칙을 따릅니다.

## 4. 파일은 어디에?
* 모든 코드는 `Assets/Scripts/` 폴더 안에서 기능별로 나누어 저장합니다.

## 5. 단계별 작업 방식
* 복잡한 기능을 만들 때는 바로 코드를 짜지 말고 **[어떻게 만들지 생각하기 -> 순서 정하기 -> 코드 쓰기 -> 확인하기]** 순서로 진행해 주세요.

## 6. AI 기반 최적화 작업 원칙 (AI Optimization)
* **에디터 스크립트화:** 반복되거나 수동 조작이 많이 필요한 씬 빌드 작업은 MCP 툴로 일일이 수행하는 대신, `Assets/Editor/` 폴더 내에 임시 유틸리티 코드를 작성하여 단번에 실행함으로써 오류를 방지하고 작업 효율을 높입니다.
* **데이터 관리:** 모든 수치(능력치 등)는 가급적 `ScriptableObject`에 정의하고, AI가 이 리포지토리만 수정하도록 유도하여 씬 손상 리스크를 최소화합니다.
