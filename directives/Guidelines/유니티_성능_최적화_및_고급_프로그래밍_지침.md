# 🚀 유니티 성능 향상 및 고급 설계 지침

이 문서는 단순한 따라하기를 넘어 시니어 급의 설계와 최적화를 하기 위한 **고급 지침**입니다.

## 1. 데이터 주도 설계 (데이터로 게임 만들기)
*   **ScriptableObject(SO) 활용**: 코드 안에 숫자나 글자를 직접 적지 않습니다. 게임 설정, 아이템 능력치 등은 ScriptableObject 파일로 분리하여 유연하게 관리합니다.
*   **이벤트 기반 구조**: 객체들끼리 너무 끈끈하게 붙어있지 않게 하여, 한 곳을 고쳤을 때 다른 곳에서 문제가 생기지 않도록 `UnityEvent` 등을 사용합니다.

## 2. 메모리 사용량 줄이기 (최적화)
*   **쓰레기 청소(GC) 관리**:
    *   `Update()` 함수 안에서 글자를 합치는 행위는 메모리 쓰레기를 많이 만드니 주의하세요. 글자를 합칠 때는 `StringBuilder`를 권장합니다.
*   **미리 만들어 두기 (Object Pooling)**: 총알이나 이펙트처럼 자주 생겼다 없어지는 물건들은 통제를 위해 미리 만들어 두는 '오브젝트 풀링'을 반드시 사용하세요.
*   **화면에 그리는 횟수 줄이기 (Batching)**: 
    *   같은 그림을 사용하는 UI 요소들은 한 번에 그려지도록 계층 구조를 잘 묶어주세요.

## 3. 유니티 화면 편하게 쓰기
*   **도구 커스터마이즈**: 복잡한 설정이 필요한 기능은 전용 관리 화면(Editor)을 만들어서 편하게 쓸 수 있게 합니다.
*   **주석과 힌트 달기**: `[Header]`, `[Range]`, `[Tooltip]` 등을 적극 활용하여 유니티 화면에서 실수 없이 작업하게 합니다.

## 4. 버그 찾기와 테스트
*   **깔끔한 로그**: 실제 게임이 완성되어 빌드될 때는 로그가 찍히지 않도록 정리합니다.
*   **자동 테스트**: 아주 중요한 핵심 기능은 수동으로 해보지 않아도 컴퓨터가 알아서 확인해 주는 테스트 기능을 사용합니다.

## 5. 작업 프로세스
1.  **생각하기**: 데이터를 어떻게 관리할지, 어떤 구조로 짤지 먼저 고민합니다.
2.  **검토하기**: 매 초마다 실행되는지, 무겁지는 않은지 미리 확인합니다.
3.  **만들기**: 앞서 정한 규칙들을 지키며 코드를 작성합니다.
4.  **확인하기**: 잘 되는지 확인하고 문제가 없는지 다시 체크합니다.
